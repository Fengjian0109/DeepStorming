# DeepStorming AI-first 工作区手动测试教程

- 适用版本：Stage 1–5 完成后的自用版
- PDF 范围：第一版仅支持可选择文字的电子版 PDF，不支持扫描件 OCR
- 安全要求：真实 API Key 只在“设置 → AI Provider → Provider 详情”的密码框输入

## 1. 启动

```bash
cd /Users/hezhendong/Desktop/deepstorming/.worktrees/codex-all-stages
pnpm install
pnpm dev
```

`pnpm dev` 会在启动前自动为 Electron 重建 SQLite 原生模块，并在退出后恢复 Node 测试所需版本。若终端仍显示 `DATABASE_UNAVAILABLE`，先退出残留 DeepStorming/Electron 进程，再确认当前目录可写；不要删除已有数据库。开发环境的 `corepack enable` 权限错误不影响已经可用的 `pnpm`。

## 2. 设置 Provider、个人资料与导师

1. 打开“设置 → AI Provider”，先在 Provider 列表点击“新增 Provider”，保存后再打开对应详情，执行连接测试并设为当前 Provider。
2. 打开“个人资料”，设置名称和头像并保存。
3. 打开“导师 / 伙伴”，先从导师列表选择已有导师或点击“新增导师”，进入独立详情页后编辑头像、性格、语气、擅长领域、书籍/论文策略和自定义指令；确认长表单可以独立滚动到底部。
4. 打开“课堂设置”，确认默认节奏、剩余上下文压缩阈值（默认 30%）和最近保留轮数。

通过标准：列表不回显 Key；重启后 Provider、个人资料、导师和课堂设置仍存在。

## 3. 导入 PDF 并开始第一节课

1. 进入“文档库”，点击带文件图标的“导入 PDF”，选择可选择文字的电子版 PDF。
2. 等待“PDF 已导入”，确认详情只显示摘要，不铺开全文。
3. 可先打开阅读器检查页面文本、block 和图片，再关闭阅读器。
4. 点击“开始课堂”，选择导师以及慢/标准/快三档之一，然后进入课堂。

通过标准：首问由当前 AI Provider 生成；未启用 Provider 时明确提示必须配置 AI，不出现本地规则导师兜底。

## 4. 检查聊天呈现

1. 确认导师和学习者头像正确；导师动作是灰色斜体，正式回答为正体。
2. 输入 `$a=\\sum_{i=1}^{N}i^2$`，发送后确认显示为公式而不是原始美元符号。
3. 检查导师回答中的 `$...$` / `$$...$$` 同样由 KaTeX 渲染。
4. 检查引用卡包含原文、页码和理由；点击来源返回 PDF 证据位置。
5. 若导师引用导入时提取的图，确认聊天显示图片、图题、页码和引用理由。

通过标准：Markdown、公式、引用和图片都位于连续对话流中；生成记录、诊断和复习任务只在“课堂信息”抽屉中。

## 5. 层级侧栏、外观、压缩与导出

1. 应用首次打开只显示主侧栏；点击“设置”打开副侧栏，再次点击同一“设置”应收起副侧栏，点击其他主导航项应切换到对应内容。
2. 主、副侧栏右上角各只有一个图标切换键。收起主侧栏时副侧栏必须一起关闭，界面保留 48px 图标轨道；再次点击轨道上的展开键恢复主侧栏。
3. 拖动侧栏与内容区之间的分隔线：默认两级侧栏合计约占窗口 40%，最大不得超过 50%；拖动条平时弱化，悬停或拖动时变清晰。
4. 打开“设置 → 外观”，分别验证浅色、深色和跟随系统；“跟随系统”应随 macOS 外观变化，重启后仍保留选择。
5. 检查文件选择器、开关和图标按钮均为统一矢量样式；按钮文案不因侧栏变窄而换行，键盘 Tab 可聚焦且有可见焦点环。
6. 长对话接近阈值后，打开“课堂信息 → 技术”，确认出现快照版本、压缩前剩余比例、阈值和最近整理状态。
7. 分别导出 Markdown 和 PDF；Markdown 应保留 LaTeX 源、引用和相对图片资源，PDF 应能离线打开并正确显示中文、公式和图片。

## 6. 下课、复习、第二节课与重启

1. 点击“下课并保存记忆”，确认后等待 AI 总结完成。
2. 选择“立即复习”，填写复习回答并完成；也可另测“先休息”，重启后应继续停留在待复习状态。
3. 回到同一教材，再开始一节新课。
4. 进入“课堂”，确认同一教材下有两条独立课程记录。
5. 完全退出并重新启动应用，确认课程消息、完成状态、记忆、复习结果、头像和 Provider 状态均恢复。

## 7. 自动化复验

```bash
pnpm check
pnpm test:e2e
pnpm package:dir
```

真实 DeepSeek 复验是显式可选操作，命令和无工件安全约束见 `docs/planning/provider-cloud-release-acceptance.md`。不要把 Key 写入命令参数、仓库文件、截图或测试报告。
