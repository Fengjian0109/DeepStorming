# DeepStorming AI-First Learning Workspace Redesign Design

- 日期：2026-07-14
- 状态：已完成产品设计确认，等待实施计划
- 适用范围：Desktop Renderer、Application、Domain、Infrastructure、Main/Preload、SQLite、DeepSeek Provider
- 依赖：现有文档/PDF 导入、Lesson 状态机、学习诊断、复习调度、论文阅读模式与阅读地图

## 1. 背景

当前 DeepStorming 已具备文档导入、PDF 文本提取、课堂会话、AI Provider、论文模式、学习诊断和复习任务等基础能力，但用户界面仍以开发阶段的信息展示为主：

- 文档库同时显示新建表单、文档列表和全文详情，导致页面过长，PDF 入口被挤到下方。
- 课堂把来源证据、消息、模型生成记录、学习诊断和复习任务依次堆叠在一个长页面中，核心对话被稀释。
- 左侧导航上下分散，缺少教材与多节课堂之间的层级关系。
- 书籍与论文课堂存在本地导师模板兜底，不符合最终产品的严格 AI 教学定位。
- Prompt、Provider、导师人格、头像和课堂偏好尚未形成统一设置中心。
- 消息仍按纯文本展示，不能可靠呈现 Markdown、LaTeX、来源引用和 PDF 图表。

本设计把 DeepStorming 重构为以 AI 对话为中心的长期学习工作区，同时保留既有证据链、状态机、诊断和复习能力。

## 2. 已确认的产品决策

1. 文档库采用顶部导入入口和两栏资源库，不默认展示全文。
2. 课堂采用具有父子层级的双侧栏与纯对话主画布。
3. 一级、二级侧栏可分别折叠，并提供一个按钮一键收起两级侧栏。
4. 两级侧栏总宽度可拖拽调整，但不得超过窗口宽度的 50%。
5. 生成记录、学习诊断、复习任务和来源证据不进入普通消息流，仅在按需信息抽屉中查看。
6. 书籍和论文课堂的教学输出均采用严格 AI 驱动；生产环境不生成本地导师回复。
7. 可创建多个导师/伙伴档案，但一节课堂只绑定一个档案。
8. 用户和导师头像可配置；导师动作使用灰色斜体，正式回答与提问使用正常字形。
9. 用户与 AI 消息都支持安全 Markdown 和 LaTeX 渲染。
10. PDF 第一版仅支持具有可选择文字层的电子版 PDF，不支持纯扫描版 PDF。
11. 第一版不让 AI 分析图片像素；AI 基于图号、图题和正文描述理解图表，并在引用时展示导入所得图片。
12. 同一本教材或论文可以对应多节课堂记录。
13. 一节课只有在 AI 下课总结保存成功并完成本节复习后才标记为完成。
14. 用户可在下课总结后选择立即复习或休息；休息后课堂保持待复习状态。
15. 每次开课前选择慢、标准、快三档节奏。
16. 聊天记录支持导出 Markdown 和 PDF。
17. 当预计剩余上下文低于用户设置的阈值时自动压缩；默认阈值为 30%，即上下文使用量接近 70% 时触发。
18. `deepseek_api.txt` 只作为后续本机真实联调凭据来源，API Key 不进入 SQLite、日志、Prompt、fixture 或 snapshot。

## 3. 目标与非目标

### 3.1 目标

- 用户导入教材或论文后，可以直接选择导师和节奏进入 AI 课堂。
- 课堂主画布始终以对话为中心，消息列表和输入框构成主要交互。
- AI 的提问、讲解、诊断、总结和复习建议均可追溯到文档证据。
- 公式、引用和图表在用户消息与 AI 消息中都能正确渲染。
- 同一文档支持多节课堂、累计学习记忆和跨课堂连续学习。
- 长对话通过可配置的 AI 上下文压缩维持可用性，同时保留完整原始历史。
- 所有外部工作具备加载、成功、错误、取消和重试语义。
- 真实 DeepSeek 可用于集成验收，而自动化测试保持确定性和可重复。

### 3.2 第一版非目标

- 不支持纯扫描版 PDF、OCR、手写公式识别或复杂版面重建。
- 不使用视觉模型分析图像像素、曲线趋势或图中未写入文本的语义。
- 不支持一节课堂中多个导师同时参与或自动轮换导师。
- 不支持语音输入、语音合成、实时协同或云同步。
- 不允许导师人格 Prompt 修改文档事实、证据边界、掌握判断规则或安全规则。
- 不在第一版实现完整主题市场、伙伴商店或在线分享社区。

## 4. 分阶段交付策略

该范围跨越多个独立子系统，不采用一次性重写。每一阶段必须形成可运行、可测试、可回退的纵向切片。

### 阶段 1：应用框架与对话式课堂

- 重构一级/二级双侧栏、折叠与拖拽行为。
- 重构文档库为顶部导入和两栏资源库。
- 把课堂改成消息流、固定输入框和按需右侧信息抽屉。
- 先复用现有 Lesson 数据，不改变 AI 生成语义。

### 阶段 2：设置中心与人物档案

- 把 Provider 移入设置中心。
- 增加用户资料、导师/伙伴档案、头像和课堂默认设置。
- 保存导师档案和 Prompt 版本，并把档案快照绑定到课堂。

### 阶段 3：严格 AI 与结构化导师回合

- 移除生产路径的本地导师回复。
- 引入结构化 `TutorTurn`、严格引用校验和一次 AI 修复请求。
- Provider 不可用时阻止开课或保留失败回合等待重试。

### 阶段 4：富消息渲染

- 统一用户与 AI 的 Markdown/LaTeX 渲染。
- 渲染导师动作、引用卡片、代码块、表格和安全链接。
- 完成接近 Claude 阅读体验的排版令牌与主题设置。

### 阶段 5：PDF 图表资产与聊天引用

- 电子版 PDF 导入时生成页面渲染图和图表资产索引。
- 通过图号、图题、页码和边界区域匹配 `FigureReference`。
- 直接图片提取失败时使用页面裁剪图，不调用本地教学兜底。

### 阶段 6：多节课堂、下课记忆、复习和上下文压缩

- 完成一本文档多节课堂的层级与生命周期。
- 增加三档节奏、下课总结、立即复习/休息和累计教材记忆。
- 增加模型感知的上下文预算与 AI 滚动压缩。

### 阶段 7：导出与真实 DeepSeek 验收

- 导出 Markdown 及资源目录。
- 导出渲染后的 PDF。
- 增加显式启用的真实 DeepSeek 集成测试与完整手动验收教程。

## 5. 信息架构

### 5.1 一级侧栏

一级侧栏是全局导航，内容顶部对齐，第一版包含：

- 文档库
- 课堂
- 设置

Provider 不再单独占用全局入口，而是位于“设置 → AI Provider”。一级侧栏可折叠为图标栏。

### 5.2 二级侧栏

二级侧栏是一级模块的上下文子级：

- 文档库：文档列表、搜索和筛选。
- 课堂：教材/论文 → 课堂记录的树形层级。
- 设置：AI Provider、导师/伙伴、个人资料、课堂、外观、数据与隐私。

二级侧栏可独立折叠。全局“收起侧栏”按钮同时收起一级和二级侧栏，再次点击恢复上一次分别保存的状态。

### 5.3 宽度规则

- 一级侧栏展开宽度固定在一个紧凑范围，折叠后只保留图标。
- 二级侧栏与主内容区之间提供可键盘访问的拖拽分隔线。
- 两级侧栏展开宽度总和不得超过当前窗口宽度的 50%。
- 窗口缩小时自动收敛二级侧栏宽度，保证主内容区仍可操作。
- 宽度和折叠状态作为非敏感 UI 偏好持久化，重启后恢复。

### 5.4 右侧信息抽屉

课堂右侧抽屉默认关闭，包含：

- 来源证据
- 学习进度
- 学习诊断
- 复习内容
- 当前 AI 模式、导师和 Prompt 版本

打开抽屉不改变消息列表的滚动锚点。生成记录、原始 Provider 错误和状态机审计只在诊断/调试入口中按需展开，不在普通课堂界面持续展示。

## 6. 文档库与阅读器

### 6.1 文档库布局

页面顶部提供：

- 粘贴文本
- 导入文件（PDF、TXT、MD）
- 搜索与筛选

粘贴正文使用对话框或独立创建页，不再常驻占据一整列。

主区域采用两栏：

- 左侧：紧凑文档列表。
- 右侧：标题、类型、页数/字数、摘要、最近学习位置、课堂数量和操作按钮。

右侧不渲染全文。用户通过“打开阅读器”查看正文或 PDF 页面，通过“开始课堂”进入开课准备。

### 6.2 PDF 支持边界

第一版仅接受具有文本层的电子版 PDF。导入流程必须检测：

- 文件是否合法 PDF。
- 是否加密或受密码保护。
- 是否存在可用文本层。
- 是否能解析页面、文本块和坐标。

纯扫描版或文本层为空时返回稳定错误码和明确中文提示，不创建看似成功但不可学习的文档。

### 6.3 导入任务

PDF 导入是持久化长任务，阶段为：

```text
queued → parsing_text → rendering_pages → extracting_figures → building_index → ready
                                                               └→ failed/cancelled
```

在开始外部或耗时工作前保存任务。每个阶段可恢复、可取消，重试不得创建重复文档、页面、文本块或图表资产。

## 7. PDF 图表资产

### 7.1 数据模型

```ts
type DocumentFigure = {
  id: string
  documentId: string
  pageNumber: number
  figureLabel: string | null
  caption: string | null
  boundingBox: { x: number; y: number; width: number; height: number }
  assetId: string
  extractionKind: 'embedded_image' | 'page_crop'
  sourceTextBlockIds: string[]
}
```

### 7.2 提取策略

1. 保存页面尺寸、文本块内容及坐标。
2. 识别 `Figure`、`Fig.`、`图` 等图题模式，并记录图号、图题和所在页。
3. 优先提取 PDF 中可定位的嵌入图片。
4. 无法安全提取单张图片时，使用页面渲染图与图题邻近区域生成裁剪图。
5. 所有文件保存到应用数据目录，数据库只保存受控资产 ID 和元数据。

第一版不声称理解图片像素。发送给文本模型的是图号、图题、页码、相关正文和稳定图表 ID。

### 7.3 聊天匹配

AI 只能从本轮上下文提供的可用图表列表中返回 `FigureReference`。Application 校验图表属于当前文档后，Renderer 才展示图片、图题、页码和“回到来源”。未知或跨文档图表 ID 使结构化回合进入修复或失败流程。

## 8. 设置中心

### 8.1 AI Provider

- Provider 类型
- 显示名称
- Base URL
- 模型名称
- API Key 状态
- 能力声明
- 连通测试、取消和稳定错误提示

API Key 只保存到 macOS Keychain。Renderer 只能获知“已配置/未配置”，不得获得明文。

### 8.2 导师/伙伴

每个导师档案包含：

- 名称与头像
- 性格描述
- 表达语气
- 擅长领域标签
- 教学严格度
- 苏格拉底追问强度
- 提示与直接讲解倾向
- 书籍教学策略
- 论文教学策略
- 用户自定义要求
- 创建时间、更新时间和版本

可以创建多个档案，但一节课堂绑定一个档案快照。修改导师后不篡改历史课堂。更换导师通过新建或复制课堂完成。

首次运行时创建一个可编辑、可复制的中性苏格拉底导师档案，确保用户无需先设计人物也能开课。系统始终至少保留一个可用导师档案；删除已被历史课堂引用的档案只会停止后续选择，不会破坏历史快照。

### 8.3 个人资料

- 用户名称
- 用户头像
- 默认显示方式

用户头像和导师头像导入后复制到应用数据目录。Preload 只暴露明确的头像导入 API，不暴露任意文件读取能力。

### 8.4 课堂设置

- 书籍默认导师
- 论文默认导师
- 默认课堂节奏
- 发送快捷键
- 自动滚动
- 上下文压缩阈值，默认剩余 30%
- 最近原文保留轮数

### 8.5 外观

- 明暗主题
- 字号和行距
- 消息最大宽度
- Claude 风格排版预设

“Claude 风格”指舒展的行距、清晰的层级、克制的边框和适合长文阅读的字体栈，不复制或分发不可用的专有字体。

### 8.6 数据与隐私

- 数据目录信息
- 导出入口
- 缓存和图表资产清理
- 删除本地学习数据

破坏性操作必须二次确认。

## 9. 严格 AI 课堂

### 9.1 生产行为

书籍和论文课堂的以下内容由 AI 生成：

- 首次提问
- 后续追问、提示和短讲解
- 教学动作选择
- 学习诊断
- 论文阶段判断和阅读地图更新
- 下课总结和课堂记忆
- 复习内容与建议
- 上下文压缩摘要

生产路径不存在本地导师回答或本地教学判断兜底。规则代码仍负责：

- 输入校验
- 状态机合法迁移
- 证据和图表所有权校验
- 上下文预算
- 幂等性
- 安全错误映射

### 9.2 Provider 不可用

- 开课前没有已配置且已激活的 Provider：阻止开课并跳转设置。
- 课堂中 Provider 请求失败：保留学习者消息和失败 ModelRun，显示重试。
- 结构化输出无效：允许一次使用同一 Provider 的修复请求。
- 修复仍失败：本轮失败，不插入伪造导师消息、引用、诊断或记忆。

### 9.3 Prompt 分层

Prompt 组装顺序固定：

1. 不可编辑的证据、安全与输出 Schema 规则。
2. 书籍或论文教学策略。
3. 导师人格和表达风格。
4. 课堂节奏。
5. 教材累计记忆与本节压缩记忆。
6. 本轮检索证据、可用图表列表和最近对话。
7. 学习者当前消息。

文档内容始终作为不可信证据数据，不得成为系统指令。

## 10. 结构化导师回合

```ts
type TutorTurn = {
  narrationMarkdown?: string
  responseMarkdown: string
  citations: CitationReference[]
  figures: FigureReference[]
  teachingAssessment: TeachingAssessment
}

type CitationReference = {
  sourceAnchorId: string
  quote: string
  rationale?: string
}

type FigureReference = {
  figureId: string
  rationale?: string
}
```

- `narrationMarkdown` 是导师动作与神态，只允许表现层内容。
- `responseMarkdown` 是正式回答、提问、讲解和公式。
- `citations` 必须指向本轮允许的来源锚点。
- `figures` 必须指向本轮允许的图表资产。
- `teachingAssessment` 驱动状态机、学习诊断和复习，不直接混入普通消息气泡。

## 11. 对话界面与富消息

### 11.1 消息布局

- 学习者消息右对齐，显示用户头像。
- 导师消息左对齐，显示绑定导师头像。
- 导师动作使用灰色斜体。
- 正式回答和问题使用正常字形。
- 不显示“导师提问”等额外区块标题。
- Prompt 版本、模型名称和状态机记录不显示在每条普通消息下。

消息列表独立滚动，输入框固定在底部。发送成功后自动定位最新消息；用户主动向上阅读历史时不强制抢回滚动位置。

### 11.2 Markdown 与 LaTeX

用户和 AI 消息使用同一个安全渲染管线：

- CommonMark/Markdown
- GFM 列表、表格和删除线
- 行内公式 `$...$`
- 块级公式 `$$...$$`
- 代码块与行内代码
- 引用块

示例 `$a=\sum_{i=1}^{N} i^2$` 在学习者发送后必须继续以数学公式显示。渲染器禁止原始 HTML 和脚本，链接协议使用白名单。

### 11.3 引用卡片

引用不只显示引号文本，而是渲染为强调卡片：

- 原文
- 文档标题
- 页码或字符范围
- 引用用途
- 回到阅读器来源位置

AI 不能提交任意页码字符串代替来源锚点。

### 11.4 图表卡片

图表卡片包含：

- 图号
- 图片或页面裁剪图
- 图题
- 页码
- 回到来源

图表紧跟引用它的导师回复展示，不作为独立系统区块堆叠。

## 12. 一本文档对应多节课堂

```text
LearningDocument 1 ── N LessonSession
LessonSession     1 ── 1 MentorSnapshot
LessonSession     1 ── N LessonMessage
LessonSession     1 ── N ContextSnapshot
LessonSession     1 ── 0..1 LessonMemory
LessonSession     1 ── N ReviewItem
```

课堂二级侧栏按文档分组展示多节课，至少显示日期、导师、节奏、消息数量和状态。

### 12.1 课堂状态

```text
preparing → active → ending → review_pending → reviewing → completed
              ├→ paused
              └→ error
```

- `preparing`：选择导师、节奏和起始位置。
- `active`：正常对话。
- `ending`：AI 正在生成下课总结和记忆。
- `review_pending`：总结已保存，用户选择休息或尚未开始复习。
- `reviewing`：正在以对话形式完成本节复习。
- `completed`：总结保存且本节复习完成。
- `paused`：用户中途暂停但尚未下课。
- `error`：可恢复的课堂工作失败。

上一节课待复习时允许新建下一节课，但开课准备页必须明显提示并提供“先复习”入口。

### 12.2 三档节奏

```ts
type LessonPace = 'slow' | 'balanced' | 'fast'
```

- 慢：小知识块、更多追问、更少直接答案。
- 标准：追问、提示与讲解平衡。
- 快：合并知识块、减少重复追问、更快进入总结与测验。

节奏在开课前选择并保存为课堂快照。它影响 Prompt 和教学动作约束，不修改导师档案。

## 13. 下课、记忆与复习

### 13.1 下课流程

1. 用户点击“下课并保存记忆”。
2. 系统持久化结束任务并禁止重复触发。
3. AI 生成结构化课堂记忆。
4. Application 校验并保存总结、掌握内容、误区、未解决问题、引用和建议。
5. 根据记忆创建本节复习内容。
6. 界面显示“开始复习”和“休息”。
7. 立即复习进入对话式复习；休息进入 `review_pending`。

### 13.2 课堂记忆

课堂记忆至少包含：

- 本节主题
- 覆盖范围
- 稳定掌握内容
- 仍不稳定内容
- 典型误区
- 关键引用和图表
- 未解决问题
- 下一节建议起点

### 13.3 教材累计记忆

完成一节课后，AI 基于既有累计记忆和本节记忆生成新版本。下一节课只加载压缩后的累计记忆，而不是全部历史消息。

学习事实、课堂证据和伙伴叙事保持分离。导师人格不能改写掌握结论或教材证据。

## 14. 上下文预算与自动压缩

### 14.1 触发规则

设置项表示“剩余上下文比例阈值”，默认 30%。当预计剩余可用上下文小于等于阈值时触发压缩，等价于使用量接近 70%。

预算必须包含：

- 核心系统规则
- 输出 Schema
- 导师和节奏设置
- 教材累计记忆
- 本节压缩记忆
- 检索证据和图表清单
- 最近原始消息
- 为模型输出保留的 Token

### 14.2 压缩语义

- 压缩由 AI 生成结构化摘要。
- 原始消息永久保留并继续用于查看和导出。
- 压缩只改变后续模型上下文组装。
- 保留最近若干轮原文。
- 公式定义、关键结论、误区、未解决问题、来源锚点 ID 和图表 ID 进入强制保留字段。
- 每个压缩版本保存覆盖的消息区间，避免重复压缩。

### 14.3 失败处理

- 压缩在一次回复成功后后台准备。
- 下一次发送时仍未完成，显示“正在整理课堂记忆”。
- 压缩失败但上下文仍安全时允许继续有限回合并提示重试。
- 接近模型硬上限时暂停生成，必须先完成压缩。
- 失败不得删除原始历史或用本地摘要替代。

## 15. 导出

### 15.1 Markdown

导出内容包括：

- 课堂元数据
- 用户与导师名称
- 消息正文
- 导师动作斜体
- LaTeX 源码
- 引用信息
- 图表相对路径
- 下课总结

图像写入同名资源目录，Markdown 使用相对路径，保证离线可读。

### 15.2 PDF

PDF 通过专用只读 HTML 打印视图生成，必须正确渲染：

- 字体与段落
- Markdown
- LaTeX
- 引用卡片
- 图表和图题
- 分页边界

### 15.3 可靠性

导出使用显式保存对话框，支持加载、成功、错误和取消。导出内容不得包含 API Key、完整系统 Prompt、内部 ModelRun 输入或调试日志。

## 16. 数据与存储

建议新增或扩展以下持久化实体：

- `learner_profiles`
- `mentor_profiles`
- `mentor_profile_versions`
- `app_settings`
- `document_figures`
- `document_assets`
- `lesson_sessions` 的导师快照、节奏和新状态字段
- `lesson_memories`
- `document_learning_memories`
- `context_snapshots`
- `lesson_end_jobs`
- `export_jobs`

API Key 继续通过 `secret_ref` 指向 Keychain。头像、页面图和图表文件位于应用数据目录，SQLite 只保存受控标识与元数据。

数据库迁移必须可前向执行，不修改或删除既有用户课堂与文档数据。

## 17. 异步、幂等与错误处理

### 17.1 通用规则

每个用户触发的异步流程都提供：

- loading
- success
- stable error
- cancellation（有意义时）
- retry

长任务在外部工作前持久化。重试写操作使用 `operationId` 或任务 ID 保证幂等。

### 17.2 AI 错误

至少区分：

- Provider 未配置
- Key 缺失或无效
- 额度不足
- 模型不存在
- 限流
- 网络错误
- 超时
- 用户取消
- 结构化响应无效
- 引用或图表越权
- 上下文超过硬上限

错误消息对用户安全，不包含 Authorization header、API Key、完整请求体或本地私密路径。

### 17.3 Markdown 与本地资源安全

- 禁止原始 HTML 和脚本。
- 链接协议白名单化。
- 本地图片只允许应用管理的资产 ID。
- Renderer 不直接接收任意 `file://` 路径。
- PDF 和文档文本作为不可信数据进入 Prompt。

## 18. 测试策略

### 18.1 Domain/Application

- 侧栏偏好值和最大宽度约束的纯逻辑测试。
- 导师档案、版本和课堂快照规则。
- 一本文档多节课堂与课堂状态迁移。
- 三档节奏映射。
- 严格 AI：无 Provider、失败、取消、修复失败。
- `TutorTurn` Schema、引用所有权和图表所有权。
- 下课任务幂等、记忆更新和复习完成规则。
- 上下文阈值、Token 预算、压缩区间和强制保留字段。
- 导出任务幂等和敏感字段排除。

### 18.2 Infrastructure

- SQLite 迁移和重启恢复。
- Keychain 读写不泄露明文。
- 电子 PDF 文本层检测。
- 页面渲染、图题识别、嵌入图片提取和页面裁剪。
- 扫描版 PDF 返回稳定不支持错误。
- DeepSeek 错误映射和取消。
- Markdown/PDF 导出文件内容与资源目录。

### 18.3 Renderer

- 双侧栏层级、分别折叠、一键折叠、拖拽和 50% 上限。
- 文档库顶部导入和紧凑详情。
- 消息对齐、头像、动作斜体和正式正文。
- 用户与 AI 的 Markdown/LaTeX。
- 引用卡片与图表卡片。
- 信息抽屉不破坏消息滚动位置。
- 开课准备、三档节奏、下课、立即复习和休息。
- 上下文压缩状态和失败恢复。
- 导出进度、取消和错误。

### 18.4 Electron E2E

至少覆盖：

1. 导入电子版 PDF。
2. 选择导师和节奏。
3. 通过真实应用边界创建课堂。
4. 提交带 LaTeX 的学习者回答并正确渲染。
5. AI 返回引用和图表并正确展示。
6. 触发上下文压缩并继续课堂。
7. 下课生成记忆。
8. 选择立即复习并完成课堂。
9. 重启后恢复教材、多节课堂、记忆和侧栏偏好。
10. 导出 Markdown 和 PDF。

### 18.5 真实 DeepSeek 验收

自动化单元与常规 E2E 使用确定性测试替身，避免网络、额度和响应波动影响持续验证。另提供显式启用的真实 DeepSeek 集成测试：

- 仅在本机手动启用。
- 从 Keychain 或安全环境注入凭据。
- 不记录 Key、Authorization header 或完整私密文档内容。
- 验证首次提问、追问、结构化引用、下课记忆和压缩摘要。

## 19. 验收标准

以下条件全部满足后，本次重构才视为完成：

1. PDF 导入入口在文档库顶部可见，电子版 PDF 可进入阅读器和课堂。
2. 文档详情不默认显示全文，不产生页面级超长滚动。
3. 一级/二级侧栏具有层级、可调宽、可分别折叠、可一键收起，总宽度不超过 50%。
4. 课堂主画布只显示对话，诊断、复习和生成记录不再堆叠在消息下方。
5. 未配置真实 Provider 时不能生成导师回复。
6. 用户与导师头像、导师人格和擅长领域可配置并绑定课堂。
7. 导师动作、正式回答、Markdown、LaTeX、引用和图表正确渲染。
8. 同一教材可查看多节课堂，完成规则符合“总结 + 复习”。
9. 三档节奏会影响 AI 教学策略。
10. 剩余上下文低于默认 30% 时自动压缩，原始历史仍完整可见。
11. 下课记忆、教材累计记忆和复习状态在重启后保持。
12. Markdown 和 PDF 导出可离线查看公式、引用和图表。
13. `pnpm check` 通过。
14. Desktop E2E 在可运行 Electron 的环境通过。
15. 真实 DeepSeek 集成验收由用户显式启用后通过。

## 20. 设计结论

DeepStorming 的核心交互从“多个开发功能区块堆叠”转为“长期、可追溯、严格 AI 驱动的学习对话”。文档、课堂、导师、记忆和复习分别拥有清晰边界；证据与图表通过稳定 ID 进入结构化 AI 输出；完整历史与压缩上下文相互分离；人物个性只影响表现与教学策略，不篡改事实和掌握判断。

实施必须按照第 4 节的阶段逐步推进，每个阶段独立编写实施计划、使用测试驱动开发、完成 `pnpm check`，并在涉及桌面集成的阶段执行 `pnpm test:e2e`。
