# DeepStorming 产品规格说明书

- 文档版本：v0.1
- 产品阶段：规划冻结稿
- 更新日期：2026-07-10
- 首发平台：macOS
- 后续平台：Windows（架构预留）
- 产品形态：本地优先的单用户桌面应用

## 1. 产品定义

DeepStorming 是一个面向教材、论文和专业知识文档的 AI 深度学习系统。它以用户手动导入的 PDF 为主要知识来源，通过苏格拉底式提问、费曼式复述、原文证据引用、迁移练习、批判性阅读和间隔复习，帮助用户形成可以解释、应用和质疑的真实理解。

DeepStorming 不是普通 PDF 阅读器，也不是“上传文件后自由聊天”的通用机器人。它的核心价值是把静态阅读材料转化为一条可诊断、可追踪、可恢复、可验证的主动学习路径。

产品定位语：

> AI 驱动的苏格拉底式深度学习与论文阅读系统。

英文定位语：

> An AI-powered system for Socratic learning, Feynman understanding, and deep paper reading.

## 2. 已确认的产品假设

1. 从空仓库重建，不复用旧版 Socratic Studio 业务代码。
2. 旧版仅作为需求原型和失败案例参考，不直接迁移数据库 Schema。
3. macOS 为首发平台，但底层不采用只能运行在 macOS 的业务实现。
4. 桌面框架采用 Electron，前端采用 React + TypeScript。
5. 所有教材、论文、学习记录和索引默认保存在本机。
6. MVP 使用 BYOK 云模型：用户配置自己的 API Key，由桌面应用直接调用模型提供商。
7. 教材学习先形成完整闭环，论文阅读复用同一文档底座后加入。
8. 教学效果优先于角色陪伴；首版大致按教学 70%、陪伴 30% 设计。
9. 不以多 Agent、复杂游戏化或云端账号系统作为 MVP 前提。

## 3. 目标用户

### 3.1 核心用户

- 需要系统学习专业教材的大学生和研究生。
- 需要深入阅读科研论文的研究人员和工程师。
- 准备考试、科研入门、面试或跨领域学习的自学者。
- 不满足于自动摘要，希望真正理解概念、公式、方法和证据的用户。

### 3.2 典型使用情境

1. 用户导入一本深度学习教材，按章节完成概念学习、复述和练习。
2. 用户导入一篇机器人或具身智能论文，理解研究动机、方法、实验与局限。
3. 用户在几天后收到复习任务，重新解释此前薄弱的概念。
4. 用户查看学习记录，知道自己在哪些概念上反复出现误区。
5. 用户点击 AI 回答中的引用，跳回 PDF 对应页并查看原文或图表。

## 4. 产品原则

### P1：证据优先

教材和论文中的事实性教学内容必须尽可能绑定来源锚点，包括文档、页码、片段和可选坐标。模型补充的文档外知识必须显式标记。

### P2：主动学习优先

默认通过问题、复述、比较、推导和迁移任务促进理解，不把长篇总结作为主要学习方式。

### P3：适时讲解

苏格拉底教学不是无限追问。用户连续卡住时，系统应进入提示阶梯，必要时提供短讲解，然后要求用户用自己的话重新组织。

### P4：掌握必须有证据

“已掌握”不能只依赖模型主观印象。系统应保存复述、迁移题、错误修复和延迟回忆等掌握证据。

### P5：无静默失败

任何用户触发的异步操作必须进入 `loading`、`success`、`error` 或 `cancelled` 之一。禁止点击后无反馈、空 `catch` 或只在开发控制台报错。

### P6：本地优先但不伪装成完全离线

PDF 原文件、索引和学习记录默认保存在本地。使用云模型时，相关教材片段和课堂上下文会发送给用户选择的模型提供商，产品必须明确告知这一点。

### P7：陪伴与教学解耦

关闭伙伴功能后，教材学习、论文阅读、掌握判断和复习系统仍须完整可用。角色语气不能修改事实判断和学习成绩。

## 5. 核心概念模型

### 5.1 LearningDocument

系统底层统一使用 `LearningDocument`，而不是将所有内容都建模为 `Book`。

文档类型包括：

- `textbook`：教材、讲义、课程资料。
- `paper`：学术论文、技术报告、预印本。
- `generic`：后续扩展的其他专业文档。

### 5.2 LearningWorkflow

不同文档类型使用不同工作流：

- `TextbookLearningWorkflow`
- `PaperLearningWorkflow`

两者共享文档解析、证据检索、课堂运行、费曼复述、学习记忆和复习基础设施，但不共用同一套课程 Prompt 和掌握规则。

### 5.3 SourceAnchor

所有可引用内容统一通过来源锚点定位：

```ts
type SourceAnchor = {
  documentId: string
  pageStart: number
  pageEnd: number
  blockIds: string[]
  chunkId?: string
  sectionPath?: string[]
  boundingBoxes?: BoundingBox[]
}
```

## 6. 核心用户流程

### 6.1 首次启动与 Provider 配置

1. 用户首次启动应用。
2. 系统解释本地数据与云模型片段发送边界。
3. 用户选择 Provider，填写模型、Base URL 和 API Key。
4. 应用安全保存 Key，并执行最小连接测试。
5. 测试成功后进入文档库；失败时展示可操作的错误说明。

### 6.2 PDF 导入

1. 用户选择本地 PDF。
2. 应用立即创建可见的导入任务。
3. 依次执行复制、校验、解析、结构化、分块和索引。
4. 用户可以查看阶段进度、取消任务或在失败后重试。
5. 导入完成后进入文档详情页；应用重启后任务状态仍可恢复。

### 6.3 教材学习

1. 用户选择教材章节或概念。
2. 系统展示本节学习目标和预计范围。
3. 课堂进行预检查和已有知识激活。
4. AI 通过苏格拉底问题推进一个核心概念。
5. 用户卡住时进入提示阶梯或微讲解。
6. 用户完成费曼式复述和迁移任务。
7. 系统保存掌握证据、误区和复习任务。

### 6.4 论文阅读

1. 用户导入论文 PDF。
2. 系统识别标题、作者、摘要、章节、图表和参考文献等结构。
3. 首先建立 `Why → What → How → Evidence → Limits → Next` 全局地图。
4. 用户选择快速速览、深度精读、公式推导、实验审查或审稿人模式。
5. AI 通过问题引导用户重构论文论证，而非只生成摘要。
6. 用户用自己的语言解释贡献、方法、实验和局限。
7. 系统形成论文理解卡片、疑问和研究启发。

### 6.5 复习

1. 首页显示到期复习任务。
2. 用户先在不看答案的情况下回忆或解释。
3. 系统根据表现和反馈更新下一次复习时间。
4. 复习记录回写到对应概念、论文观点或误区。

## 7. 功能需求

### 7.1 AI Provider

- `PROV-001` 支持 Mock Provider，用于无网络开发和确定性测试。
- `PROV-002` 支持 DeepSeek Provider。
- `PROV-003` 支持 OpenAI-compatible 自定义接口。
- `PROV-004` 支持新增、编辑、删除、启用和测试连接。
- `PROV-005` API Key 不得以明文进入 Renderer 或日志。
- `PROV-006` 掩码字符串不得覆盖真实 Key。
- `PROV-007` 区分认证失败、连接失败、限流、余额或配额、模型不存在和响应格式错误。
- `PROV-008` Provider 能力可声明流式输出、结构化输出、Embedding 和视觉能力。

### 7.2 文档导入与管理

- `DOC-001` 支持手动选择 PDF。
- `DOC-002` 导入任务必须持久化状态和进度。
- `DOC-003` 使用文件哈希识别重复导入。
- `DOC-004` 保存页级文本和布局块。
- `DOC-005` 保存图像、图表、公式区域等可定位资产。
- `DOC-006` 检测无有效文本的扫描 PDF，并明确提示 OCR 状态。
- `DOC-007` 支持取消、重试和失败原因查看。
- `DOC-008` 支持删除文档及其派生索引，同时保护其他文档数据。
- `DOC-009` 索引必须可以从原始页和块重新构建。

### 7.3 阅读器与检索

- `READ-001` 支持页码跳转、缩放、搜索和目录导航。
- `READ-002` AI 引用可以跳转并高亮对应原文。
- `READ-003` 支持查看模型使用过的全部证据片段。
- `READ-004` 首版采用词法检索和章节过滤，保留混合向量检索扩展口。
- `READ-005` 检索结果包含页码、章节、分数和邻接上下文。
- `READ-006` 低质量文本可以回退显示原页或局部截图。

### 7.4 教材课堂

- `TUTOR-001` 课堂使用显式状态机，不由单个长 Prompt 隐式控制。
- `TUTOR-002` 每轮默认只推进一个主要知识点和一个主要问题。
- `TUTOR-003` 支持提示阶梯、短讲解和回到主线。
- `TUTOR-004` 支持临时问题支线，并记录从何处离开和返回。
- `TUTOR-005` 支持暂停、恢复和异常中断恢复。
- `TUTOR-006` 教材事实必须附带来源锚点或明确标记来源不足。
- `TUTOR-007` 课堂结束前完成费曼复述或显式选择稍后完成。
- `TUTOR-008` 保存课堂步骤、消息、引用、评价和状态转换。

### 7.5 费曼评价与掌握判断

- `ASSESS-001` 复述按正确性、完整性、因果性、清晰度和迁移性评价。
- `ASSESS-002` 评价必须指出具体正确部分、缺失点和错误依据。
- `ASSESS-003` 用户可以重试复述并比较前后变化。
- `ASSESS-004` 掌握度由多条证据更新，而不是单一总分覆盖。
- `ASSESS-005` 延迟回忆结果可以修正此前掌握判断。

### 7.6 论文阅读

- `PAPER-001` 支持论文元数据与章节结构。
- `PAPER-002` 支持研究问题、核心贡献、方法、实验、证据、局限和未来方向的结构化表示。
- `PAPER-003` 支持快速速览、深度精读、公式推导、实验审查和审稿人模式。
- `PAPER-004` 建立“观点—证据”关系，避免实验和结论脱节。
- `PAPER-005` 支持图、表、公式和 Caption 的来源定位。
- `PAPER-006` 输出论文理解卡片、未解决疑问和研究启发。
- `PAPER-007` MVP 后阶段支持单篇论文；多论文比较不阻塞首版。

### 7.7 学习记忆与复习

- `MEM-001` 分离学习事实、课堂事件、会话摘要和伙伴叙事记忆。
- `MEM-002` 记录概念掌握证据和常见误区。
- `MEM-003` 会话摘要可以追溯到原始课堂步骤。
- `REV-001` 支持到期复习队列。
- `REV-002` 复习调度器通过接口隔离，首版算法可以被 FSRS 等实现替换。
- `REV-003` 保存每次复习的提示、答案、评分、耗时和下一次计划。

### 7.8 伙伴体验

- `COMP-001` 伙伴是可选教学表现层，不是教学事实来源。
- `COMP-002` 伙伴语气和教学策略分别配置。
- `COMP-003` 叙事记忆与掌握状态分表保存。
- `COMP-004` 首版只提供原创、克制的人设模板，不复制受版权保护的角色。

## 8. 教学状态机

```text
PRECHECK
→ ACTIVATE_PRIOR_KNOWLEDGE
→ SOCRATIC_PROBE
→ HINT_LADDER
→ MICRO_EXPLANATION
→ FEYNMAN_TEACH_BACK
→ MISCONCEPTION_REPAIR
→ TRANSFER_CHALLENGE
→ MASTERY_DECISION
→ REFLECTION
→ COMPLETED
```

允许的关键回路：

- 复述失败：`FEYNMAN_TEACH_BACK → MISCONCEPTION_REPAIR → SOCRATIC_PROBE`
- 前置知识缺失：创建补救支线，完成后返回原状态。
- 用户主动暂停：任何可中断状态进入 `PAUSED`，恢复后回到保存的状态。
- 模型或网络失败：保留最后一个已提交状态，重试不得重复写入消息或评价。

## 9. 非功能需求

### 9.1 稳定性

- 用户动作应立即获得视觉反馈，长任务不得阻塞界面线程。
- 导入、索引和课后整理必须支持幂等重试。
- 应用异常退出后不得破坏已完成的文档和课堂数据。
- 数据库变更必须通过顺序迁移执行。

### 9.2 安全与隐私

- Renderer 禁止直接访问 Node.js、SQLite 和文件系统。
- API Key 使用操作系统加密能力保护。
- 日志自动清除 Key、Authorization Header 和完整敏感请求体。
- 发送给云模型前，界面必须能说明发送内容的类型和范围。

### 9.3 可维护性

- Domain 不依赖 React、Electron、SQLite 或具体模型 SDK。
- 所有 IPC、Provider 输出和持久化 DTO 使用运行时 Schema 校验。
- Prompt 必须版本化，不得散落在组件文件中。
- 核心模块具备单元测试，端到端关键路径具备自动化测试。

### 9.4 可访问性与国际化

- MVP 界面以中文为主，数据结构不得假设文档语言为中文。
- 关键操作支持键盘访问和可辨识焦点状态。
- 颜色不能成为错误、成功或掌握度的唯一表达方式。

## 10. MVP 范围

MVP 完整闭环：

```text
配置模型
→ 导入一份文本型 PDF 教材
→ 完成解析和检索
→ 查看 PDF
→ 开始一节教材课堂
→ 获得可跳转引用
→ 完成费曼复述
→ 保存课堂和误区
→ 生成并完成一次复习
```

MVP 不包含：

- 云账号、云同步和计费系统。
- 移动端。
- 实时语音和角色动画。
- 多人课堂。
- 自动网络搜索和自动下载论文。
- 多 Agent 编排。
- 大规模知识图谱可视化。
- 完全离线大模型。

## 11. 产品级验收标准

1. 新用户能够完成 Provider 配置并获得明确的成功或错误结果。
2. 导入按钮永远不会出现“点击后无反应”。
3. 一份测试教材可以从导入一路完成到复习，应用重启后数据仍然存在。
4. 教材事实性回答包含可跳转到原页的引用。
5. 用户连续答不上来时，系统会帮助而不是无限追问。
6. 费曼评价能够指出具体缺失点，并支持修改后重试。
7. 任何 Provider、解析或数据库错误都具有稳定错误码和用户可理解文案。
8. API Key 不出现在 Renderer 状态、数据库明文字段或日志中。
9. 核心闭环通过自动化端到端测试后，才能进入论文和伙伴阶段。

## 12. 成功指标

首版不追求虚假的“学习效率提升三倍”宣传，先收集可验证指标：

- PDF 导入成功率及失败原因分布。
- 从导入到第一次课堂的完成率。
- 单节课完成率和主动中断率。
- 首次复述与修复后复述的差异。
- 延迟复习正确率。
- 引用跳转使用率。
- 用户对“是否真正理解”的主观评价。
- 模型调用成本、延迟和结构化输出失败率。

## 13. 术语表

- **BYOK**：Bring Your Own Key，用户配置自己的模型 API Key。
- **来源锚点**：能把模型内容定位回文档页、块和坐标的结构。
- **教学动作**：课堂状态机在一轮中执行的提问、提示、讲解或评价。
- **掌握证据**：支持或反驳“用户已掌握某知识点”的可追溯记录。
- **误区**：用户反复出现或对后续理解有影响的错误模型。
- **学习工作流**：针对教材或论文组织教学过程的业务规则集合。
